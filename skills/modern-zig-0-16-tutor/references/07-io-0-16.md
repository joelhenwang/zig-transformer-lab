# 07 ‚Äî I/O as an Interface (0.16.0 std.Io)

The single biggest 0.16.0 change. Everything that blocks control flow or
introduces nondeterminism now takes an `io: std.Io` parameter. No more
`std.io.getStdOut()`. No more generic reader/writer. Buffers and flushing are
explicit.

## Mental model

- `std.Io` is a vtable + userdata pair ‚Äî like `Allocator`.
- `main` picks an implementation (`Io.Threaded`, `Io.Evented`, `Io.Uring`, ‚Ä¶)
  and threads `io` through the program.
- `std.Io.Reader` and `std.Io.Writer` are **concrete interfaces**, not
  generics. The buffer lives in the interface, user-supplied at construction.
- `GenericReader`, `AnyReader`, `FixedBufferStream` ‚Äî **deleted**.

## Minimal hello-world (0.16)

```zig
const std = @import("std");

pub fn main(init: std.process.Init) !void {
    try std.Io.File.stdout().writeStreamingAll(init.io, "Hello, World!\n");
}
```

`writeStreamingAll` takes `io` and a byte slice, handles short writes, no
buffering needed for one-shot output.

## Buffered writer (for frequent small writes)

```zig
pub fn main(init: std.process.Init) !void {
    var buf: [4096]u8 = undefined;
    var fw = std.Io.File.stdout().writer(init.io, &buf);

    try fw.interface.print("x = {d}\n", .{42});
    try fw.interface.print("y = {d}\n", .{43});

    try fw.flush(); // REQUIRED ‚Äî tail bytes sit in `buf` otherwise
}
```

- `fw.interface` is the `std.Io.Writer` to call `.print` / `.writeAll` on.
- `try fw.flush()` pushes the user buffer to the underlying file.
- Forgetting `flush()` is the #1 "nothing printed" bug.

## Unbuffered writer

```zig
var fw = std.Io.File.stdout().writer(init.io, &.{}); // empty buffer
try fw.interface.writeAll("immediate\n"); // writes straight through
```

## Reader with a fixed buffer

```zig
const data: []const u8 = "a,b,c";
var r: std.Io.Reader = .fixed(data);
const byte = try r.takeByte(); // 'a'
_ = byte;
```

`std.Io.Reader.fixed(slice)` replaces `std.io.fixedBufferStream`. For files:

```zig
const file = try std.Io.Dir.cwd().openFile(init.io, "in.txt", .{});
defer file.close(init.io);
var fr = file.reader(init.io, &.{});   // unbuffered
const bytes = try fr.interface.allocRemaining(init.arena.allocator(), .limited(1 << 20));
_ = bytes;
```

> VERIFY WITH ZIG 0.16.0 LOCALLY the exact namespace of `.limited(n)`. The
> release notes use `.limited(1234)` against both `readFileAlloc` and
> `allocRemaining`.

## `std.debug.print` ‚Äî when it's OK

`std.debug.print` writes to **stderr** and does not require `io`. Use it for:

- Ad-hoc debugging while iterating.
- Programs without `init: std.process.Init` (simple `pub fn main() void`).
- Panic / fatal messages.

Do NOT use it in library code that should be routed through `io`.

## File read / write patterns

```zig
// Read whole file (new 0.16 API)
const contents = try std.Io.Dir.cwd().readFileAlloc(
    init.io,
    "input.txt",
    init.arena.allocator(),
    .limited(1 << 20),   // 1 MiB cap, returns error.StreamTooLong if exceeded
);

// Write text file
const f = try std.Io.Dir.cwd().createFile(init.io, "out.txt", .{});
defer f.close(init.io);
var buf: [4096]u8 = undefined;
var fw = f.writer(init.io, &buf);
try fw.interface.print("value = {d}\n", .{42});
try fw.flush();
```

**Mentor note.** Error rename: `FileTooBig` ‚Üí `StreamTooLong`.

## `File.Stat.atime` is now optional

```zig
const stat = try file.stat(init.io);
const at = stat.atime orelse return error.FileAccessTimeUnavailable;
_ = at;
```

Some filesystems don't expose access time.

## Common review findings

- `std.io.getStdOut().writer()` ‚Äî stale, flag always.
- `std.io.bufferedWriter(...)` ‚Äî stale, flag always.
- `std.io.fixedBufferStream` ‚Äî stale, flag always.
- `GenericReader` / `AnyReader` / `GenericWriter` / `AnyWriter` ‚Äî stale, flag.
- Missing `try fw.flush()` before end of scope on a buffered writer ‚Äî flag.
- `std.debug.print` in a function that takes `io` ‚Äî prefer the `io` route.
- `file.readToEndAlloc(a, n)` ‚Äî replaced by `file.reader(io, &.{}).interface.allocRemaining(a, .limited(n))`.

## Common mentor diagnostic questions

- "Where does `io` come from? The `init` in `main`, or is this library code?"
- "Is this buffered? If so, where do we flush?"
- "What's the upper bound on the stream? (The `.limited` argument is not
  optional by convention.)"
- "Could this be a simple `writeStreamingAll` instead of a buffered writer?"
